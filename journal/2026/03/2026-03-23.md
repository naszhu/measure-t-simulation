## network issue (saved in claude conv mar23 as well)
Hardware: ThinkPad P16 Gen 3, Intel I226-V ethernet (igc driver) + Intel BE200
   WiFi (iwlwifi driver).                                                       
                                                                                
  Problem: After suspend (lid close), both network adapters fail to reinitialize
   on resume. The root cause is the igc driver hitting a Timeout reading        
  IGC_PTM_STAT register during the suspend phase, leaving the ethernet in a     
  broken state. WiFi similarly fails to come back. NetworkManager is running
  fine — it just has no working hardware underneath it.

  Why reboot fixed it: A full reboot reinitializes all hardware from scratch,   
  bypassing the broken resume path.
                                                                                
  Why restarting the network service didn't fix it: The problem is at the kernel
   driver level, not the network service level. Restarting NetworkManager does
  nothing when the underlying kernel drivers are stuck.                         
                  
  Fix: A systemd sleep hook that forcibly unloads and reloads both drivers on   
  every resume, giving them a clean re-initialization.

  ---                                                                           
  1. Created /lib/systemd/system-sleep/reload-network-drivers.sh (new file,
  executable)                                                                   
                  
  Created twice — first version had a bug in the unload order, second overwrote 
  it with the corrected version. Final content:                                 
  #!/bin/bash
  case $1 in                                                                    
    post)         
      rmmod iwlmld 2>/dev/null
      rmmod iwlwifi 2>/dev/null
      rmmod igc 2>/dev/null    
      sleep 1
      modprobe igc                                                              
      modprobe iwlwifi
      ;;                                                                        
  esac            

  ---
  2. Manually reloaded kernel modules (runtime only, no persistent change)
                                                                                
  - Unloaded: iwlmld, iwlwifi, igc
  - Reloaded: igc, iwlwifi                                                      
                  
  This was just to restore networking in the current session — it has no        
  persistent effect and will not repeat unless the sleep hook triggers it.
                                                                                
  ---             
  3. Brought up enp134s0 (runtime only, no persistent change)
                                                                                
  Ran ip link set enp134s0 up to bring the wired interface up after the driver
  reload. Again, runtime only.                                                  
                  
  ---                                                                           
  That's everything. The only persistent change is the sleep hook file. Nothing
  was changed in GRUB, NetworkManager config, network profiles, or any other    
  system config.
