### Botvinick \& Plaut (2006) vs. Traditional Weight-Based Models

| Aspect                 | **Traditional (Weight-Based)**                             | **Botvinick \& Plaut (Activation-Based)**                            |
| ---------------------- | ---------------------------------------------------------- | -------------------------------------------------------------------- |
| **Core equation**      | $W_{t+1} = W_t + f_t \otimes \psi_t$                       | $h_t = f(W_{ih}x_t + W_{hh}h_{t-1})$                                 |
| **Memory carrier**     | Weight matrix $W$ (stored associations)                    | Hidden state $h_t$ (evolving activation)                             |
| **Space type**         | Tensor \/\ Product space $\mathcal{F} \otimes \mathcal{C}$ | Hilbert (Inner-product) space $\mathcal{H}$                          |
| **Encoding mechanism** | Add outer products → accumulate new bindings               | Update state via fixed function $F_W$ → move in activation manifold  |
| **Update meaning**     | Write new components into $W$ (memory grows)               | Shift internal position $h_t$ (memory flows)                         |
| **Temporal dynamics**  | Context drift drives additive trace writing                | Continuous trajectory governed by recurrent operator                 |
| **Rationale**          | Memory = structure built over time (trace storage)         | Memory = instantaneous state within fixed dynamics (state evolution) |
### Mathematical Forms: Traditional vs. Botvinick \& Plaut

#### **1. Traditional Weight-Based (Context / Product-Space Regime)**

- **Encoding (outer-product binding):**  
  $W_{t+1} = W_t + f_t \otimes \psi_t$

- **Context drift:**  
  $\psi_{t+1} = A\psi_t + Bf_t$

- **Retrieval (projection):**  
  $a(i|\text{cue}) = f_i^{\top} W \psi_{\text{cue}}$

- **Interpretation:**  
  Memory grows by accumulating rank-1 bindings $f_t \otimes \psi_t$ in tensor space $\mathcal{F} \otimes \mathcal{C}$;  
  temporal change = expansion of $W$ along new binding directions.


#### **2. Botvinick \& Plaut (2006) Activation-Based (Hilbert Regime)**

- **Recurrent update (state evolution):**  
  $h_t = f(W_{ih}x_t + W_{hh}h_{t-1})$

- **Output decoding:**  
  $y_t = \text{softmax}(W_{ho}h_t)$

- **Training (backpropagation through time):**  
  $\Delta W = -\eta \frac{\partial E}{\partial W}, \quad E = -\sum_j t_j \log y_j$

- **Interpretation:**  
  The operator $F_W$ defines a continuous mapping $h_{t+1} = F_W(h_t, x_t)$ in Hilbert space $\mathcal{H}$;  
  memory = trajectory $\{h_t\}$ generated by this fixed vector field, not additive weight storage.


#### **3. Structural Contrast (Unified Operator Perspective)**

| Regime | Representation | Core Equation | Memory Dynamics |
|----------|----------------|----------------|----------------|
| Product-space | $W \in \mathcal{F}\otimes\mathcal{C}$ | $W_{t+1} = W_t + f_t \otimes \psi_t$ | Additive construction (trace writing) |
| Hilbert-space | $h_t \in \mathcal{H}$ | $h_{t+1} = F_W(h_t, x_t)$ | Continuous state evolution (trajectory) |
